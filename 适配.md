# AnchorCAF 下游服务器 (SubServer) 适配指南

本文档旨在指导开发者如何将自己的服务（SubServer）接入 AnchorCAF 开放平台。

## 1. 接入流程概述

1.  **自动注册**：SubServer 启动时调用 AnchorCAF 的注册接口，自动获取 `Client ID` 和 `Client Secret` 并持久化保存。
2.  **用户授权**：通过 OAuth 2.0 协议引导用户授权，获取用户的 `Access Token`。
3.  **数据交互**：使用 `Access Token` 与 AnchorCAF 中心服务器交互，获取权限、接口列表或其他 SubServer 的访问令牌。
4.  **服务间通信**：使用从 AnchorCAF 获取的令牌访问其他 SubServer 的受保护资源。

## 2. 注册 SubServer

### 2.1 准备工作
你需要生成一对 RSA 密钥（公钥和私钥）。公钥将在注册时提交给 AnchorCAF，私钥由你自行保管，用于后续可能的签名验证或解密操作。

### 2.2 注册接口
SubServer 首次启动时，应调用以下接口进行注册：

*   **URL**: `POST http://<CAF_SERVER_HOST>/api/subserver/register`
*   **Content-Type**: `application/json`
*   **Body**:
    ```json
    {
      "name": "My Awesome App",
      "public_key": "-----BEGIN PUBLIC KEY-----..."
    }
    ```
*   **Response**:
    ```json
    {
      "id": "YOUR_CLIENT_ID",
      "secret": "YOUR_CLIENT_SECRET",
      "message": "Registered successfully"
    }
    ```

> **注意**：请务必将返回的 `id` (Client ID) 和 `secret` (Client Secret) 安全地持久化保存。后续的 OAuth 流程和 API 调用都需要使用这些凭证。不要重复注册。

## 3. 用户授权 (OAuth 2.0)

AnchorCAF 使用标准的 OAuth 2.0 授权码模式 (Authorization Code Flow)。

### 3.1 获取 Authorization Code

引导用户访问以下 URL 进行授权：

```
GET http://<CAF_SERVER_HOST>/web/oauth/authorize?client_id=<YOUR_CLIENT_ID>&redirect_uri=<YOUR_CALLBACK_URL>
```

*   `client_id`: 注册时获得的 Client ID。
*   `redirect_uri`: 用户授权后的回调地址。

用户同意授权后，浏览器将跳转回 `redirect_uri`，并附带 `code` 参数：
`<YOUR_CALLBACK_URL>?code=AUTHORIZATION_CODE`

### 3.2 换取 Access Token

使用获得的 `code` 向 AnchorCAF 请求 Access Token：

**Endpoint**: `POST /api/oauth/token`

**Content-Type**: `application/json`

**Body**:
```json
{
  "grant_type": "authorization_code",
  "client_id": "YOUR_CLIENT_ID",
  "client_secret": "YOUR_CLIENT_SECRET",
  "code": "AUTHORIZATION_CODE"
}
```

**Response**:
```json
{
  "access_token": "eyJhbGciOiJSUzI1NiIs...",
  "token_type": "Bearer",
  "expires_in": 3600
}
```

## 4. API 接口参考

所有受保护的接口都需要在 Header 中携带 Access Token：
`Authorization: Bearer <ACCESS_TOKEN>`

### 4.1 获取用户权限
查询当前用户授予该 SubServer 的权限列表。

*   **URL**: `GET /api/protocol/permissions`
*   **Response**:
    ```json
    {
      "permissions": [
        { "id": "read_profile", "description": "Read User Profile" }
      ]
    }
    ```

### 4.2 获取接口列表
获取当前用户可用的所有 SubServer 接口信息。

*   **URL**: `GET /api/protocol/interfaces`
*   **Response**:
    ```json
    {
      "sub_servers": [
        {
          "id": "...",
          "name": "Another App",
          "interfaces": ["/api/v1/data", "/api/v1/status"]
        }
      ]
    }
    ```

### 4.3 获取目标服务访问令牌
如果你需要访问另一个 SubServer 的数据，需要先向 AnchorCAF 申请针对该目标的访问令牌。

*   **URL**: `POST /api/protocol/access-token`
*   **Body**:
    ```json
    {
      "target_sub_server_id": "TARGET_SUBSERVER_ID"
    }
    ```
*   **Response**:
    ```json
    {
      "access_token": "eyJ..." // 这是一个专门用于访问目标 SubServer 的 Token
    }
    ```

### 4.4 更新自身接口列表
告知 AnchorCAF 你提供了哪些接口，以便其他 SubServer 发现。

*   **URL**: `POST /api/protocol/subserver/interfaces`
*   **Body**:
    ```json
    {
      "interfaces": ["/api/user/info", "/api/posts/list"]
    }
    ```

### 4.5 更新黑名单
阻止特定的 SubServer 访问你的数据。

*   **URL**: `POST /api/protocol/subserver/blocklist`
*   **Body**:
    ```json
    {
      "block_list": ["BAD_SUBSERVER_ID"]
    }
    ```

## 5. WebSocket 事件订阅

SubServer 可以连接 WebSocket 以实时接收权限变更通知。

*   **URL**: `ws://<CAF_SERVER_HOST>/api/ws`
*   **Header**: `Authorization: Bearer <ACCESS_TOKEN>` (连接时通过 Header 或 Query 参数传递 Token)

**接收消息格式**:
```json
{
  "type": "permission_update",
  "payload": { ... }
}
```

## 6. 常见问题

*   **Token 过期**: Access Token 有效期默认为 24 小时（可配置）。过期后需要重新引导用户授权或使用 Refresh Token（如有实现）。
*   **安全性**: 请务必验证从 AnchorCAF 获取的 Token 的签名。AnchorCAF 的公钥可以通过公开渠道获取或在注册时交换。
